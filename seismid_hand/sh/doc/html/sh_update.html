Update the source code of SH/SHM to the most recent version

The procedure described here let you update the source code of your existing SH installation to the the most recent version. It uses the version numbers in the beginning of every file in the SH/SHM distribution. You get a copy of my currently used source files if a newer version is available here. Please note that you need an already existing installation which is going to be upgraded. Perform the following steps:

  • Make a version list of your own source files using vrs_list.csh. Call it with the SH-root directory as argument and save the output into a file:
    chmod a+x vrs_list.csh
    vrs_list.csh $SH_ROOT >$HOME/myvrslist.txt
    Please choose descriptive names for your files to avoid interfering with files of other users. Do not create the version list file in $SH_ROOT or one of its subdirectories in order to avoid recursive version lists.
  • Upload your result file (in this example: myvrslist.txt) to our ftp server: ftp://ftp.szgrf.bgr.de/pub/incoming/shupd.
  • Request file updates for your version file Please enter the name of your version file here and then click on this button:.
  • Copy the update tarfile to $SH_ROOT and call install_ftp_update.csh from there:
    cp myvrslist.tar.gz $SH_ROOT
    cd $SH_ROOT
    install_ftp_update.csh myvrslist.tar.gz
